# 强化学习基础：理解 V, Q 和 Advantage

## 先建立直觉：用"下棋"类比

强化学习的场景可以抽象成：

```
一个智能体（Agent）在环境中不断做决策，
每次决策后环境给出反馈（reward），
目标是最大化长期累积 reward。
```

对应到 LLM：
```
Agent     = 语言模型
环境      = 人类偏好 / 规则评分器
决策      = 每次生成一个 token
Reward    = 最终对整个回复的打分
```

---

## 核心概念体系

### 1. 基本元素

| 符号 | 名称 | 含义 | LLM 对应 |
|------|------|------|---------|
| `s` | State（状态） | 当前所处的情境 | prompt + 已生成的 token |
| `a` | Action（动作） | 在当前状态下的决策 | 生成下一个 token |
| `r` | Reward（奖励） | 执行动作后的即时反馈 | 最终打分（稀疏） |
| `π` | Policy（策略） | 给定状态，选择动作的规则 | 语言模型本身 |
| `γ` | Discount factor | 对未来 reward 的折扣系数 | 通常 0.99 |

### 2. Return（回报）：累积 reward

单步 reward 不够用，RL 真正关心的是**从当前时刻开始，未来所有 reward 的总和**：

```
G_t = r_t + γ·r_{t+1} + γ²·r_{t+2} + γ³·r_{t+3} + ...
```

γ 的作用：越远的 reward 折扣越大，体现"近期回报比远期确定"的直觉。

```
γ=0.9 时：
  t+1 的 reward 权重 = 0.9
  t+2 的 reward 权重 = 0.81
  t+5 的 reward 权重 = 0.59
  t+10 的 reward权重 = 0.35
```

---

## V 函数：一个状态"值多少钱"

### 定义

```
V^π(s) = 从状态 s 出发，按策略 π 行动，期望能拿到的 G_t
```

用人话说：**"我现在在这个状态，按我的策略走下去，平均能得多少分？"**

### 下棋类比

```
开局（s_0）：V = 0.5    （五五开）
你走出好棋（s_5）：V = 0.8   （你占优）
对手走出神之一手（s_8）：V = 0.2  （你处于劣势）
```

V 值随局面变化而变化，反映当前形势的好坏。

### LLM 里的 V

```
s_t = "今天天气真的很____"（已生成到这里）

V(s_t) ≈ 0.7   意思是：从这个上下文继续生成，
                 预计最终能得到 0.7 分的 reward
```

### Critic 就是学 V 函数的模型

Critic 是一个神经网络，输入当前状态 `s_t`，输出预测的 `V(s_t)`。

它通过大量的生成轨迹（序列）来学习：
> "历史上在类似上下文下，最终平均得了多少分"

---

## Q 函数：一个（状态+动作）值多少钱

```
Q^π(s, a) = 在状态 s 执行动作 a，然后按策略 π 走下去，期望能拿到的 G_t
```

和 V 的区别：**V 是"这个状态"的价值，Q 是"这个状态下做这个动作"的价值**。

```
V(s) = 在 s 下按策略行动的平均价值
Q(s,a) = 在 s 下强制执行动作 a 后的价值

两者关系：
V(s) = Σ_a π(a|s) · Q(s,a)   （对所有可能动作加权平均）
```

---

## Advantage：这个动作比平均水平好多少

这是理解 PPO 的最关键概念。

### 定义

```
A^π(s, a) = Q(s, a) - V(s)
```

### 直觉

```
V(s)    = 在状态 s 下，按正常策略行动，平均能得多少
Q(s,a)  = 在状态 s 下，强制执行动作 a，能得多少
A(s,a)  = 执行动作 a，比"随便走走"好了多少
```

- `A > 0`：这个动作比平均水平**好**，应该强化
- `A < 0`：这个动作比平均水平**差**，应该抑制
- `A = 0`：这个动作和平均水平一样，无需调整

### 为什么不直接用 Q，要用 Advantage？

因为不同状态的绝对价值差异很大：

```
开局状态：Q(好棋) = 0.8,  Q(差棋) = 0.6   → 差距 0.2
残局状态：Q(好棋) = 0.99, Q(差棋) = 0.3   → 差距 0.69
```

用 Advantage 归一化之后，才能公平地比较不同状态下动作的相对好坏，**梯度更稳定**。

---

## Bellman 方程：V 的自洽条件

V 函数满足一个递推关系：

```
V(s_t) = r_t + γ · V(s_{t+1})
```

含义：**当前状态的价值 = 即时 reward + 折扣后的下一状态价值**

这是 RL 里最核心的等式。Critic 的训练本质就是让预测的 V 满足这个等式。

### 用它估计 Advantage

```
A(s_t, a_t) = Q(s_t, a_t) - V(s_t)
            = r_t + γ·V(s_{t+1}) - V(s_t)   ← 代入 Bellman
            = δ_t                              ← 这就是 TD error
```

**TD error 就是 Advantage 的单步估计**，这就是上文提到的公式的来源。

---

## 概念关系总图

```
环境给出稀疏 Reward
         ↓
G_t = 折扣累积 Reward（Return）
         ↓
    ┌────┴────┐
    V(s)     Q(s,a)
  状态价值   状态-动作价值
    └────┬────┘
         ↓
   A(s,a) = Q - V
     动作的相对好坏
         ↓
   PPO 用 A 来更新 Policy：
   强化 A>0 的 token
   抑制 A<0 的 token
```

---

## 回到 LLM 的完整图景

```
生成序列：x → y_1 → y_2 → y_3 → ... → y_T
                                           ↓
                                     Reward = 0.8

Critic 估计每步的 V：
  V(x)          = 0.5
  V(x,y_1)      = 0.55
  V(x,y_1,y_2)  = 0.6
  ...
  V(x,...,y_T)  = 0.8

计算每步的 Advantage（TD error）：
  A_1 = r_1 + γ·V(s_2) - V(s_1) = 0 + 0.99·0.6 - 0.55 = +0.044
  A_2 = r_2 + γ·V(s_3) - V(s_2) = ...
  A_T = r_T + 0 - V(s_T)        = 0.8 - 0.8 = 0

PPO 更新：
  A_t > 0 → 增大生成 y_t 的概率
  A_t < 0 → 减小生成 y_t 的概率
```

Critic 的核心价值就在于：把末端的一个分数，通过 Bellman 方程逐步往前归因，让每个 token 都知道"自己对最终结果贡献了多少"。
